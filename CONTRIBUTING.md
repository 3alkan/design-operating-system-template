# Contributing

Thanks for improving this template.

## How To Propose Changes
1. Open an issue using the templates in `.github/ISSUE_TEMPLATE/`.
2. Describe the problem, the proposed doc/template change, and expected impact.
3. Submit a PR using `.github/PULL_REQUEST_TEMPLATE.md`.

## Scope Rules For Contributions
- This repository is docs-first and template-focused.
- No product/application code in contributions.
- Keep content implementation-agnostic and developer-focused.
- Preserve placeholders (for example `[PROJECT_NAME]`, `[PROJECT_DESCRIPTION]`, `[MAINTAINER_EMAIL]`) unless intentionally replacing template tokens.

## Docs Style Rules
- Be concise and practical.
- Use Markdown only for docs.
- Use Mermaid-only diagrams in fenced ` ```mermaid ` blocks.
- Keep diagrams small; split by concern.
- Allowed Mermaid types: `flowchart`, `sequenceDiagram`, `stateDiagram-v2`.

## PR Checklist
- [ ] Change is documentation/meta/template only.
- [ ] Relevant docs are updated (PRD, architecture, ADR, runbook) when structure changes.
- [ ] Placeholders are preserved or clearly replaced.
- [ ] Any diagram follows Mermaid-only policy.
- [ ] PR template sections are completed.

## Templates
- Use issue templates for Feature, Bug, ADR proposal, and Spike.
- Use the PR template and complete the `Plan` and doc impact sections.
