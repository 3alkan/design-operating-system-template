# Changelog

All notable changes to Flow Approval Hub are documented in this file.

## Version Semantics
- `INSTANCE_DESIGN_VERSION` would track the maturity of the design package for this product.
- `PRODUCT_VERSION` would track the maturity of the implemented product.
- This example keeps both design and implementation history in one changelog because the product stays in one repo.

## [0.1.0-design]
### Design Changes
- Defined the approval workflow, policy evaluation boundary, fulfillment boundary, and audit model end to end.
- Added accepted ADRs for workflow state control and append-only audit plus additive contracts.
- Completed the quality, operations, and traceability artifacts for implementation handoff.

### Implementation Changes
- No implementation changes are included in this frozen reference example.
