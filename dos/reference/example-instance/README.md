# Reference Product Example

This example shows a completed downstream instance of the DOS for a work intake and approval platform.

It is frozen DOS reference and teaching material for humans and LLMs. It demonstrates how the DOS can be filled out, but it does not define downstream product scope by default and is never copied into instance roots by default.

## Product Summary
- Product: Flow Approval Hub
- Purpose: capture internal work requests, route them through policy-based approval, issue fulfillment tasks, and keep a durable audit trail and operator view.
- Audience: developers and LLMs learning what a completed, implementation-ready, stack-independent design package looks like.

## Why This Example Exists
- Demonstrates the quality bar for a completed instance.
- Shows how goals, scenarios, capabilities, contracts, architecture, ADRs, quality, and operations connect.
- Shows the intended same-repo lifecycle: `design/` as source of truth and `implementation/` as the post-Gate-2 code location.
- Serves as a benchmark for manual DOS review and future DOS changes.
