# PLANS.md

Use an ExecPlan for multi-hour or high-risk work.

An ExecPlan must be understandable by a newcomer with only:
- the current working tree, and
- the plan file itself.

## ExecPlan Required Sections
- Goal
- Non-goals
- Steps
- Risks
- Verification
- Rollback
- Doc updates

## Minimum Standard
- Steps are concrete and ordered.
- Verification is observable and repeatable.
- Rollback is practical, not theoretical.
- Doc updates list exact files to change.
